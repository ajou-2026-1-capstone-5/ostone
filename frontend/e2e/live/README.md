# 라이브 스모크 e2e

운영 배포본(`https://app.ajou-cstone.com`)을 브라우저로 직접 열어 **실제 백엔드를 호출**하는 라이브 스모크 테스트다.

## 목적과 기존 mock e2e와의 차이

| 구분 | mock e2e (`e2e/*.spec.ts`) | 라이브 스모크 (`e2e/live/*.live.spec.ts`) |
| --- | --- | --- |
| 대상 | 로컬 preview 서버 + 모든 API mock(`page.route`) | 배포된 운영 백엔드(실제 호출) |
| 검증 범위 | UI 흐름 + 프론트 로직 | 운영 환경이 실제로 살아 있고 핵심 흐름이 동작하는지 |
| 실행 | CI 포함 자동 실행 | **로컬 수동 전용(CI 금지)** |
| 데이터 | 메모리상 mock | **운영 실데이터에 영향** (오염 방지가 최우선) |

라이브 스모크는 mock을 일절 쓰지 않는다. 배포 직후 운영 환경의 핵심 경로(로그인, 워크스페이스, 도메인팩 조회 등)가 실제로 동작하는지 확인하는 용도다.

## 선행 작업 (운영자)

라이브 스모크는 운영 데이터에 접근하므로, 실행 전에 운영자가 다음을 준비해야 한다.

1. **권한이 최소화된 전용 테스트 계정 발급** — 운영 데이터를 광범위하게 읽거나 변경할 수 없는, 라이브 스모크 전용 계정을 발급한다. 일반 운영자/관리자 계정을 재사용하지 않는다.
2. **자격증명 주입** — 발급한 계정의 이메일/비밀번호를 실행자에게 안전한 경로로 전달한다. 평문 자격증명은 저장소에 절대 커밋하지 않는다.

## 실행법

### 1. 환경변수 파일 작성

예시 파일을 복사해 실제 값을 채운다. `.env.e2e.local`은 gitignore되며 커밋되지 않는다.

```bash
cp .env.e2e.example .env.e2e.local
```

`.env.e2e.local`에 채울 값:

| 변수 | 설명 |
| --- | --- |
| `E2E_BASE_URL` | 대상 배포 URL. 운영 대상이면 `https://app.ajou-cstone.com`. |
| `E2E_CONFIRM_PROD` | 운영 host 대상일 때 오실행 방지 확인값. 운영이면 `app.ajou-cstone.com`을 정확히 입력. 운영이 아니면 비워둔다. |
| `E2E_EMAIL` | 위에서 발급한 전용 테스트 계정 이메일. |
| `E2E_PASSWORD` | 전용 테스트 계정 비밀번호. |
| `E2E_ALLOW_POLLUTING` | @pollutes tier 활성화 여부. 기본 `false`. (아래 참고) |

### 2. 기본 실행 (정리 가능/read-only tier)

```bash
pnpm e2e:live
```

기본 실행은 데이터가 잔존하는 `@pollutes` 시나리오를 자동으로 제외한다. 포함되는 시나리오는 read-only 조회이거나, 생성하더라도 archive API로 즉시 정리되는 것들뿐이다.

## tier 구조

오염 방지를 위해 시나리오를 2단계로 나눈다.

- **기본 tier** — read-only 조회 또는 정리 가능한 쓰기(워크스페이스 생성 후 archive). `pnpm e2e:live`로 실행된다.
- **@pollutes tier** — 정리 API가 없어 데이터가 운영에 **영구 잔존**하는 쓰기(회원가입, 상담/데모 세션). 기본 실행에서 제외되며, 명시적 opt-in일 때만 실행된다.

### @pollutes tier 실행

```bash
E2E_ALLOW_POLLUTING=true pnpm e2e:live:pollutes
```

`E2E_ALLOW_POLLUTING=true`로만 활성화된다. 이 값이 없으면 spec 최상단 가드가 전부 skip된다.

> **경고: @pollutes 시나리오는 운영 데이터를 영구 잔존시킨다.**
> 회원가입 계정, 상담/데모 세션·메시지는 삭제 API가 없어 운영 DB에 그대로 남는다. 실행 여부를 신중히 판단하라.

### 잔존 데이터 수동 정리 (운영자)

@pollutes 시나리오가 생성한 데이터는 자동 정리되지 않으므로 운영자가 수동으로 정리한다.

- 라이브 스모크가 만든 데이터는 모두 **`e2e-` prefix**로 식별된다(예: 가입 이메일 `e2e-<timestamp>@example.com`, 이름 `e2e-<timestamp>`).
- 운영 관리 도구에서 `e2e-` prefix로 검색해 해당 계정/세션을 정리한다.
- 워크스페이스 자동 정리가 누락된 경우 `e2e/live/.cleanup/leftovers.log`에 잔존 리소스가 기록되니, 이 로그를 확인해 수동 정리한다.

## CI 금지 / 로컬 수동 전용

라이브 스모크는 운영 백엔드를 실제로 호출하므로 **CI에서 실행할 수 없다.** `CI` 환경변수가 설정되어 있으면 globalSetup이 실행을 차단한다. 반드시 로컬에서 수동으로만 실행한다.
