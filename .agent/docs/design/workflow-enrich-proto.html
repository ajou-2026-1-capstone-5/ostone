5 · Before vs After

현재 ostone(흑백) → 제안 시안 — 핵심 변화 요약

Before · ostone

  • · 흑백 도형(원/사각형/마름모/사다리꼴)만 사용
  • · 노드는 라벨 한 줄만 표시 (description/icon/badge 없음)
  • · Decision 분기 라벨이 패스 위에 직접 떠 가독성↓
  • · Answer/Handoff clip-path 때문에 그림자 잘림
  • · 배경 격자(gap=20) 톤이 무채색

After · enriched

  • · 카테고리별 색상 코드(violet/indigo/amber/sky/rose/zinc)
  • · lucide 아이콘 + 헤더/본문/푸터 3단 카드 위계
  • · 엣지 라벨이 pill 뱃지(좌측 색점으로 yes/no 분기 식별)
  • · clip-path 폐기 → border-radius 카드 + box-shadow 깔끔
  • · dot grid(gap=24) + 옅은 청회색 캔버스로 카드 부각

6 · Mock data schema

node.data 확장안 (graph_json 노드 객체에 옵셔널 필드 추가)

{
  "id": "n3",
  "type": "ACTION",
  "label": "VIP 등급 처리",
  "policyRef": "PR-014",

  // ↓ 신설 (모두 optional, backward-compatible)
  "description": "우선순위 높음으로 설정하고 상담 태그 VIP를 적용합니다.",
  "iconHint": "Zap",                          // lucide icon key — 미지정 시 type 기본
  "badges": ["tag:VIP", "priority:high"],
  "accentColor": "indigo",                    // violet|indigo|amber|sky|rose|zinc
  "meta": {                                   // 자유 필드
    "channel": "kakao",
    "team": "VIP 전담",
    "sla": "5m"
  }
}

Frontend → GraphNode 타입에 위 5필드를 옵셔널로 추가. Backend는 graph_json JSONB 그대로 passthrough(@JsonRawValue)이므로 DTO/DDL 변경 없음. ML은 GraphNodeSpec + serialize_graph_json 에 같은 필드 추가 필요(별도 PR).
