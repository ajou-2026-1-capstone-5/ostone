# 2-1-1. ML Runtime v2 기반 Intent Discovery / Workflow Entry Point 생성

## 목적

상담 로그를 conversation 단위로 정제한 뒤, role-aware semantic representation과 flow signature를 함께 사용해 Domain Pack 초안의 intent/workflow entry point를 생성한다.

기존 외부 임베딩 API 호출 방식은 폐기한다. 임베딩은 `ml-embedder-task` 또는 local runtime contract를 통해 생성하고, LLM은 구조화·라벨링·검증 보조에만 사용한다.

## Pipeline

```text
ingestion
→ preprocessing
→ representation
→ intent_discovery
→ flow_splitting
→ draft_generation
→ evaluation
→ publish_candidate
```

## Runtime Components

| Component | 책임 |
| --- | --- |
| `ml-pipeline-runner` | Airflow stage orchestration, manifest, schema validation |
| `ml-embedder-task` | conversation embedding batch 생성 |
| `ml-llm-service` | vLLM 기반 internal OpenAI-compatible 구조화 보조 |
| `artifact-store` | local artifact, checksum, lineage, idempotency 관리 |

## Runtime Config

| Env | 기본값 | 설명 |
| --- | --- | --- |
| `ML_ARTIFACT_STORE` | `local` | 현재 구현은 `local`만 지원한다. `s3`는 후속 artifact-store adapter에서 도입한다. |
| `ML_ARTIFACT_BUCKET` | 없음 | 후속 S3 adapter용 예약 설정 |
| `ML_ARTIFACT_PREFIX` | 빈 값 | 후속 S3 adapter용 예약 설정 |
| `EMBEDDING_MODEL_NAME` | `BAAI/bge-m3` | embedding runtime model name |
| `LLM_MODEL_NAME` | `Qwen/Qwen3-14B` | LLM runtime model name |
| `LLM_RUNTIME_BASE_URL` | 없음 | internal LLM service endpoint |
| `LLM_RUNTIME_API_KEY` | 없음 | internal LLM service key |
| `ML_RUNTIME_PROFILE` | `cheap` | `cheap`, `balanced`, `quality` |
| `GPU_TASK_MODE` | `run_task` | `run_task` 또는 `service` |

## Stage: representation

입력:

- `preprocessing/preprocessed_data.json`

처리:

- customer-only text embedding
- full-dialogue text embedding
- resolution-focused text embedding
- agent-proxy text embedding
- role-aware weighted semantic vector 생성
- flow signature sidecar 저장

기본 가중치:

```text
0.45 * customer_embedding
+ 0.20 * full_dialogue_embedding
+ 0.20 * resolution_embedding
+ 0.15 * agent_embedding
```

출력:

- `semantic_embeddings.npy`
- `embedding_index.json`
- `flow_signatures.json`
- `representation_report.json`
- `manifest.json`

## Stage: intent_discovery

기본 mode:

- `PIPELINE_INTENT_DISCOVERY_MODE=graph_leiden`

처리:

1. representation artifact가 있으면 `semantic_embeddings.npy`를 사용한다.
2. 없으면 local/runtime-backed embedder contract로 직접 embedding을 생성한다.
3. semantic vector와 flow signature를 결합한다.
4. cosine kNN graph를 생성한다.
5. Leiden community detection을 수행한다.
6. small cluster/outlier를 분리한다.
7. exemplar conversation과 cluster metric을 산출한다.

기본 파라미터:

| Parameter | 기본값 |
| --- | --- |
| `k_neighbors` | `20` |
| `min_cluster_size` | `8` |
| `leiden_resolution` | `1.0` |
| `exemplar_count` | `5` |

출력:

- `clusters.json`
- `embeddings.npy`
- `manifest.json`

## Stage: flow_splitting

처리:

- semantic cluster 내부에서 workflow entry point 후보를 분할한다.
- 현재 구현은 `ended_status`와 workflow signal을 기준으로 안정적인 subflow를 분리한다.
- 최소 분할 크기 미만의 subflow는 residual/mixed group으로 병합한다.

출력:

- `clusters.json`
- `workflow_entrypoints.json`
- `flow_split_report.json`
- `manifest.json`

## Artifact Manifest v2

모든 stage manifest는 기존 snake_case 필드와 함께 v2 필드를 포함한다.

필수 v2 필드:

- `schemaVersion`
- `stageName`
- `pipelineJobId`
- `datasetId`
- `runId`
- `inputArtifactRefs`
- `outputArtifactRefs`

- `runtimeProfile`
- `recordCount`
- `checksum`
- `createdAt`

## Evaluation Gate

`evaluation` stage는 publish 전에 privacy/schema/evidence/graph gate를 적용한다.

차단 조건:

- workflow graph에 terminal node가 없음
- workflow graph edge가 존재하지 않는 node를 참조함
- evidence coverage가 0.80 미만
- PII redaction 실패 감지
- LLM schema validity가 0.95 미만
- evidence 없는 policy/risk가 자동 확정 상태

evaluation 실패 시 `publish_candidate`는 callback을 보내지 않고 Airflow task를 실패시킨다.

## LLM Policy

허용:

- cluster name 제안
- intent description 정리
- slot/policy/risk 문장 구조화
- workflow node/edge label 정리
- evidence summary 생성
- draft/evidence contradiction check

금지:

- clustering 자체
- intent 단독 판정
- workflow graph 창작
- PII 원문 입력
- evaluation gate 우회
- 사람 승인 없는 Domain Pack publish

## 검증 기준

- production runtime 코드에 외부 임베딩 API 설정이 남아 있지 않다.
- `ML_ARTIFACT_STORE=s3`는 실제 S3 adapter가 구현되기 전까지 fail-fast로 거부된다.
- `representation → intent_discovery → flow_splitting` stage가 Airflow DAG에 연결되어 있다.
- 모든 manifest가 artifact schema v2 필드를 가진다.
- 평가 실패 candidate는 publish되지 않는다.
- `uv run pytest`, `uv run ruff check .`, `uv run ruff format --check .`, `uv run mypy .`가 통과한다.
