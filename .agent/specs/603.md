# 루트 개발 태스크 러너 제공

## Goal

루트 `package.json`의 package scripts를 개발자가 가장 자주 쓰는 로컬 개발 태스크의 공통 진입점으로 사용해 backend/frontend/ML/Docker Compose 명령을 한 곳에서 실행할 수 있게 한다.

## Problem

루트 package scripts에는 CI와 포맷 중심 명령이 존재하지만, 전체 로컬 스택 실행, 전체 품질 검사, OpenAPI 및 frontend API 재생성, 최소 구동 확인, 주요 서비스 로그 확인을 한눈에 실행하는 상위 명령이 없다. 신규 개발자는 `backend/`, `frontend/`, `ml/`, Docker Compose 명령을 각각 기억해야 해서 온보딩 비용이 커진다.

## Scope

- 루트 `package.json`에 다음 상위 package scripts를 추가한다.
  - `dev`: `.env` 준비 여부를 확인하고 backend JAR 패키징 후 Docker Compose 기반 전체 로컬 스택을 실행한다.
  - `check`: backend/frontend/ML 품질 검사를 루트에서 순차 실행한다.
  - `codegen`: `.env` 준비 여부를 확인하고 backend OpenAPI 문서 생성 후 frontend Orval API 생성을 실행한다.
  - `smoke`: `.env` 준비 여부를 확인하고 Docker Compose 주요 서비스가 running 상태인지 검증한다.
  - `logs`: `.env` 준비 여부를 확인하고 backend/frontend/Airflow 주요 서비스 로그를 확인한다.
- 기존 backend/frontend/ML 세부 package scripts는 보존하고, 상위 명령은 하위 scripts에 위임한다.
- README의 로컬 개발 명령을 루트 package scripts 기준으로 정리한다.
- AGENTS의 Root Scripts 표를 새 루트 태스크 기준으로 갱신한다.

## Non-goals

- Makefile, justfile, Taskfile 등 별도 태스크 러너를 도입하지 않는다.
- backend/frontend/ML 애플리케이션 코드, 테스트 코드, API 계약, database schema를 변경하지 않는다.
- CI workflow, branch protection, lint-staged 정책을 변경하지 않는다.
- `frontend/src/shared/api/generated/` 산출물을 재생성하지 않는다.

## Affected Paths

- `package.json`
- `README.md`
- `AGENTS.md`
- `.agent/specs/603.md`

## Requirements

1. 루트에서 `pnpm run dev`, `pnpm run check`, `pnpm run codegen`, `pnpm run smoke`, `pnpm run logs`를 확인할 수 있어야 한다.
2. backend/frontend/ML의 기존 세부 명령은 삭제하거나 의미를 바꾸지 않아야 한다.
3. `check`는 하위 명령 이름이 `check:backend`, `check:frontend`, `check:ml`로 드러나도록 위임해 실패 모듈을 추적하기 쉬워야 한다.
4. `codegen`은 `.env` 확인 뒤 `backend/build/openapi.json` 생성과 `frontend/src/shared/api/generated/` 재생성의 순서를 보장해야 한다.
5. README는 신규 개발자가 루트 package scripts를 우선 진입점으로 사용할 수 있게 주요 명령과 기존 모듈별 명령의 관계를 설명해야 한다.

## Data/API Impact

- API, persistence, runtime configuration 변경은 없다.
- OpenAPI와 frontend generated API는 `pnpm run codegen` 실행 시에만 갱신되며, 이번 변경 자체에서는 generated 산출물을 커밋하지 않는다.

## Acceptance Criteria

- `pnpm run` 출력에서 루트 개발 태스크 러너 명령을 확인할 수 있다.
- `pnpm run dev`는 `.env` 확인과 backend JAR 패키징 후 `docker compose up -d`를 실행한다.
- `pnpm run check`는 backend, frontend, ML 검사를 순차 실행하며 실패한 하위 script 이름을 출력한다.
- `pnpm run codegen`은 `.env` 확인 후 backend OpenAPI 생성과 frontend `api:gen`을 순서대로 실행한다.
- `pnpm run smoke`는 주요 Docker Compose 서비스가 running 상태가 아니면 실패해야 한다.
- README의 로컬 개발환경 섹션이 루트 `pnpm run dev/check/codegen/smoke/logs`를 기준으로 정리된다.

## Validation

- `node -e 'JSON.parse(require("fs").readFileSync("package.json", "utf8")); console.log("package.json ok")'`
- `pnpm run --silent`
- `pnpm run format:check`
- `pnpm exec prettier --check .agent/specs/603.md`
- `git diff --check`
- 필요 시 `.env`와 Docker 실행 환경에서 `pnpm run smoke`

## Open Questions

- 없음. 이슈 본문과 사용자 지시(`package.json scripts로 처리`)가 태스크 러너 선택과 구현 범위를 결정하기에 충분하다.
