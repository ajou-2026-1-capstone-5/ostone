# [Mixed] 496 - 워크스페이스 무료 온보딩 권리

## Goal

신규 워크스페이스가 활성 구독 없이도 최초 1회 raw-file 업로드부터 domain pack generation 및 review 진입까지 경험할 수 있도록 워크스페이스 단위 무료 온보딩 권리를 제공한다.

## Scope

- `app.workspace` 컬럼으로 무료 온보딩 상태를 관리한다.
- 활성 구독이 없는 워크스페이스는 무료 온보딩 상태가 허용할 때만 raw-file 업로드와 domain pack generation을 진행할 수 있다.
- raw-file 업로드가 무료 온보딩을 시작하면 해당 dataset을 온보딩 대상 dataset으로 기록한다.
- 무료 온보딩으로 시작한 domain pack generation pipeline job이 최종 상태(`SUCCEEDED`, `FAILED`, `CANCELLED`)에 도달하면 무료권을 소진한다.
- `SUPER_ADMIN`은 실패 등으로 소진된 무료권을 복구할 수 있다.
- 업로드 화면은 무료 온보딩 가능, 진행 중, 소진 상태를 표시한다.

## Non-Goals

- 유저 단위 무료권 또는 복수 무료권 지급은 구현하지 않는다.
- 자동 결제 전환, 카드 등록 유도 고도화, 요금제/결제 UX 변경은 범위 밖이다.
- quota 한도 자체를 재설계하지 않는다. 무료 온보딩 상태는 기존 quota validator가 첫 업로드/실행 allowance를 판단할 때 참조할 수 있도록 연결한다.

## Backend Design

### Workspace Free Onboarding State

`app.workspace`에 다음 컬럼을 추가한다.

| Column | Type | Description |
| --- | --- | --- |
| `free_onboarding_status` | `varchar(50) not null default 'AVAILABLE'` | `AVAILABLE`, `IN_PROGRESS`, `CONSUMED` |
| `free_onboarding_dataset_id` | `bigint null` | 무료 온보딩으로 업로드된 dataset id |
| `free_onboarding_pipeline_job_id` | `bigint null` | 무료 온보딩으로 실행된 domain pack generation job id |
| `free_onboarding_started_at` | `timestamptz null` | 무료 온보딩 시작 시각 |
| `free_onboarding_consumed_at` | `timestamptz null` | 무료권 소진 시각 |

`Workspace` 도메인은 public setter 없이 다음 상태 전이를 제공한다.

- `startFreeOnboarding(datasetId, now)`: `AVAILABLE`에서 `IN_PROGRESS`로 전환하고 dataset id를 기록한다.
- `attachFreeOnboardingPipelineJob(datasetId, pipelineJobId)`: 진행 중인 무료 온보딩 dataset에 pipeline job id를 연결한다.
- `consumeFreeOnboarding(pipelineJobId, now)`: 연결된 무료 온보딩 pipeline job이 최종 상태가 되면 `CONSUMED`로 전환한다.
- `restoreFreeOnboarding()`: `CONSUMED` 또는 실패한 진행 상태를 `AVAILABLE`로 복구하고 dataset/job/timestamp를 초기화한다.

### Entitlement Rule

새 애플리케이션 서비스는 워크스페이스별 entitlement를 다음 순서로 판단한다.

1. 현재 구독이 있고 `Subscription.isActive()`이면 raw-file 업로드와 generation trigger를 허용한다.
2. 구독이 없거나 활성 상태가 아니면 무료 온보딩 상태를 확인한다.
3. `AVAILABLE`이면 첫 raw-file 업로드만 허용하고 업로드 성공 후 `IN_PROGRESS`로 전환한다.
4. `IN_PROGRESS`이면 기록된 dataset에 대한 generation trigger만 허용한다.
5. `CONSUMED`이면 업로드와 generation trigger를 모두 차단한다.

차단 시 `BadRequestException` 계열 응답 코드 `FREE_ONBOARDING_UNAVAILABLE`을 사용한다.

### Pipeline Consumption

- `ReceiveWorkflowDraftCallbackUseCase`의 최종 workflow callback이 `PipelineJob.markSucceeded(...)`를 호출한 뒤 무료권을 소진한다.
- `ReceivePipelineJobFailureCallbackUseCase`가 active job을 `FAILED`로 종료할 때 무료권을 소진한다.
- Airflow trigger 실패로 생성 직후 `FAILED`가 되는 경우도 동일하게 소진된다.
- 이미 소진된 상태에서 중복 callback을 받아도 멱등하게 유지한다.

## Frontend Design

`WorkspaceUploadPage`는 현재 workspace를 조회해 `LogUploadForm`에 무료 온보딩 상태 필드를 전달한다. `LogUploadForm`은 기존 업로드와 generation CTA 흐름을 유지하면서 다음 상태 copy를 표시한다.

| Status | UI Behavior |
| --- | --- |
| `AVAILABLE` | 구독 없이도 첫 업로드와 초안 생성을 진행할 수 있음을 표시 |
| `IN_PROGRESS` | 무료 온보딩 진행 중임을 표시하고 동일 flow의 generation/review 이동을 강조 |
| `CONSUMED` | 활성 구독이 없으면 업로드와 generation CTA를 비활성화하고 구독 필요 안내 표시 |

새 UI는 `frontend/DESIGN.md` 기준의 흑백 chrome, 6~8px card radius, 기존 `Button`/toast 패턴을 유지한다.

## Acceptance Criteria

- workspace당 무료 온보딩은 1회만 시작된다.
- 무료 온보딩으로 시작한 dataset이 아닌 다른 dataset의 generation trigger는 활성 구독 없이는 차단된다.
- 무료 온보딩 pipeline job이 `SUCCEEDED`, `FAILED`, `CANCELLED` 최종 상태에 도달하면 workspace 무료권은 `CONSUMED`가 된다.
- `SUPER_ADMIN`은 admin endpoint로 무료권을 `AVAILABLE` 상태로 복구할 수 있다.
- 무료권 소진 후 활성 구독이 없으면 raw-file 업로드와 domain pack generation trigger가 모두 차단된다.
- 업로드 화면은 무료 온보딩 가능, 진행 중, 소진 상태를 구분해 보여준다.
