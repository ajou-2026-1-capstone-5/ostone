# 상담 처리 요약 지표

## Goal

워크스페이스 대시보드에서 선택 기간의 상담 처리량, 첫 응답 속도, 처리 주체, 미종료 세션, 전 기간 대비 증감률을 고객용 KPI 카드로 확인할 수 있게 한다.

## 범위

- `backend/src/main/java/com/init/workflowruntime/presentation/ConsultationMetricsController.java`
  - `GET /api/v1/workspaces/{workspaceId}/consultation/metrics`가 선택 기간 쿼리 파라미터를 받을 수 있게 한다.
  - 기존 워크스페이스 멤버십 검증은 유지한다.
- `backend/src/main/java/com/init/workflowruntime/application/ConsultationMetricsService.java`
  - 선택 기간과 동일 길이의 전 기간을 기준으로 지표를 비교한다.
  - 평균 계산이 불가능한 값은 `null`로 반환한다.
- `backend/src/main/java/com/init/workflowruntime/infrastructure/persistence/JpaConsultationMetricsRepository.java`
  - `runtime.chat_session`, `runtime.chat_message` 기반 집계를 확장한다.
  - 운영 상담 지표에서 demo/simulation 채널을 제외한다.
- `frontend/src/features/consultation/api/consultationApi.ts`
  - 대시보드가 `from`, `to` 기간 쿼리로 상담 지표 API를 호출할 수 있게 한다.
  - 기존 무파라미터 호출은 실시간 상담 화면 호환을 위해 유지한다.
- `frontend/src/pages/workspace/ui/WorkspaceDashboardPage.tsx`
  - #517에서 만든 대시보드 셸의 기간 필터와 KPI 카드를 연결한다.
  - 로딩, 오류, 데이터 없음, 일부 데이터 상태를 표시한다.

## 제외 범위

- workflow별 랭킹
- 자동화 커버리지 분석
- 고객 액션 추천
- 상담 만족도 지표
- domain pack version, channel, workflow status 필터의 서버 집계 반영

## API

### `GET /api/v1/workspaces/{workspaceId}/consultation/metrics`

| Query | 필수 | 설명 |
| --- | --- | --- |
| `from` | no | 선택 기간 시작일. ISO date. `to`와 함께 있을 때만 사용한다. |
| `to` | no | 선택 기간 종료일. ISO date. 포함 범위로 해석한다. |

`from`/`to`가 없으면 기존처럼 오늘 하루 기준 지표를 반환한다. `from`과 `to` 중 하나만 있거나 `from > to`이면 validation error로 응답한다.

### Response 필드

| 필드 | 설명 |
| --- | --- |
| `totalConsultationCount` | 선택 기간에 시작된 운영 상담 수 |
| `completedConsultationCount` | 선택 기간에 완료 또는 해결 처리된 상담 수 |
| `averageFirstResponseSeconds` | 고객 첫 메시지 이후 첫 응답까지의 평균 초 |
| `averageLlmFirstResponseSeconds` | LLM 첫 응답 평균 초 |
| `averageHumanFirstResponseSeconds` | 상담사/상담원 첫 응답 평균 초 |
| `llmHandledCount` | 상담사 메시지 없이 LLM이 처리한 완료 상담 수 |
| `humanInterventionCount` | 상담사/상담원 메시지가 포함된 완료 상담 수 |
| `unresolvedSessionCount` | 선택 기간에 시작됐고 아직 완료되지 않은 상담 수 |
| `comparison` | 동일 길이의 전 기간 대비 증감률. 계산 불가 시 개별 필드는 `null` |

기존 실시간 상담 상단 지표 호환을 위해 `handledTodayCount`, `llmHandledTodayCount`, `humanHandledTodayCount`는 같은 응답에 유지한다.

## 데이터 기준

- 운영 상담 지표는 `runtime.chat_session.workspace_id`와 요청 워크스페이스를 기준으로 제한한다.
- `channel`이 `DEMO`, `DEMO_WEB`, `SIMULATION`, `SIMULATION_WEB`, 또는 `SIMULATION%`인 세션은 제외한다.
- 첫 고객 메시지는 `runtime.chat_message.sender_role in ('USER', 'CUSTOMER')` 중 기간 내 세션의 가장 이른 메시지다.
- LLM 응답은 `sender_role = 'ASSISTANT'`로 본다.
- 상담사 응답은 `sender_role in ('COUNSELOR', 'AGENT')`로 본다.
- `COMPLETED` 세션은 `ended_at`이 선택 기간에 있으면 완료 처리로 본다.
- `RESOLVED` 세션은 `meta_json.resolution.resolvedAt`이 선택 기간에 있으면 완료 처리로 본다.
- `OPEN`, `ACTIVE` 세션은 선택 기간에 시작됐으면 미종료 세션으로 본다.

## UI 요구사항

- `/workspaces/{workspaceId}/dashboard`는 선택 기간의 총 상담 수, 처리 완료 상담 수, 평균 첫 응답 시간, LLM 첫 응답 시간, 상담사 첫 응답 시간, LLM 처리 건수, 상담사 개입 건수, 미종료 세션 수를 KPI 카드로 표시한다.
- 각 KPI 카드는 전 기간 대비 증감률을 함께 표시한다.
- 평균 또는 증감률 계산이 불가능하면 `--`로 표시한다.
- API 로딩 중에는 카드 값에 로딩 상태를 표시한다.
- API 오류 시 오류 패널과 toast error를 표시한다.
- 데이터가 모두 0이거나 평균이 계산되지 않아도 레이아웃은 유지한다.

## Acceptance Criteria

1. 선택 기간의 총 상담 수가 표시된다.
2. 선택 기간의 처리 완료 상담 수가 표시된다.
3. 평균 첫 응답 시간이 표시된다.
4. LLM 첫 응답 시간과 상담사 첫 응답 시간이 분리 표시된다.
5. LLM 처리 건수와 상담사 개입 건수가 분리 표시된다.
6. 미종료 세션 수가 표시된다.
7. 전 기간 대비 증감률이 표시된다.
8. 평균 계산이 불가능한 경우 `--`로 안전하게 표시된다.
9. simulation/demo 채널 세션은 운영 상담 KPI에 포함되지 않는다.
10. 워크스페이스 멤버가 아닌 사용자의 API 요청은 거부된다.

## Validation

- Backend service test: 선택 기간, 전 기간 비교, 평균 `null`, 워크스페이스 멤버십 거부를 검증한다.
- Backend repository test: `runtime.chat_session` / `runtime.chat_message` 기반 집계와 demo/simulation 제외를 검증한다.
- Backend controller test: 기본 호출과 `from`/`to` 쿼리 전달을 검증한다.
- Frontend API test: 무파라미터 metrics 호출과 기간 쿼리 호출을 검증한다.
- Frontend page test: KPI 카드, 기간 필터 요청, `--` fallback, empty/loading/error panel을 검증한다.
