# [fix/842] GPU LLM domain candidate generation timeout and lifecycle guard

## Goal

Airflow domain candidate generation should complete against the GPU-backed local LLM instead of failing from an undersized client timeout or accidental LLM service shutdown.

## Background

`pipeline_job_46` reached the internal GPU LLM service after the model task became healthy on ECS. The Qwen request then failed in `domain_candidate_generation` with `httpx.ReadTimeout` after the stage client canceled the request at 20 seconds while the llama.cpp inference server was still producing Qwen tokens. The retry investigation also showed that replay cleanup can stop the LLM ECS service during an initial run when the cleanup task is cleared or scheduled together with initial-run tasks.

After the client request stopped sending a small hardcoded `max_tokens` cap, `pipeline_job_51` exposed the next runtime boundary: the internal LLM ALB returned `504 Gateway Time-out` at its default 60 second idle timeout while llama.cpp was still decoding. ECS events also showed the LLM service target deregistering before Airflow's LLM cleanup task, which means Terraform/CD can race with Airflow's on-demand desired-count changes and scale the service back to zero during a run.

After the ALB timeout and desired-count ownership fixes, the initial pipeline runs completed and `pipeline_job_53_domain_confirmed_replay` reached `feedback_candidate_generation`. Review question enrichment then failed on the fourth LLM call because its stage-local HTTP client still used a 30 second timeout. The first three enrichment calls completed with `200 OK`, so a single slow question should not fail the whole replay; the stage should preserve the original generated question as fallback and continue. Review question enrichment also still had a hardcoded client-side `max_tokens` cap, which should be removed so the LLM server/runtime remains the single generation-budget owner.

## Scope

| Area | Change |
|------|--------|
| Domain candidate stage | Increase the default LLM read timeout and bound the generated response size. |
| Feedback candidate stage | Increase review-question enrichment timeout, remove its hardcoded client-side `max_tokens` cap, and convert per-question LLM request failures into explicit question fallback instead of stage failure. |
| Prompt contract | Request JSON-only output with no reasoning or markdown wrapper. |
| ECS stage task | Forward domain-candidate and review-question LLM runtime knobs to one-shot stage task containers. |
| Airflow DAG | Prevent replay LLM cleanup from stopping the initial-run LLM service. |
| Infrastructure | Persist Airflow ECS security-group access to the internal LLM ALB on port 8000, keep the internal LLM ALB idle timeout above the client read timeout, and prevent Terraform from resetting Airflow-owned runtime desired count. |
| Tests | Cover timeout defaults, payload caps, ECS env forwarding, and initial-run cleanup guard behavior. |

## Non-goals

- Changing the domain classification algorithm beyond LLM request reliability.
- Changing the selected Qwen model or model storage layout.
- Changing GPU ASG size, quota, or instance type selection.
- Changing frontend review rendering.

## Acceptance Criteria

- Domain candidate LLM calls default to a longer timeout than the previous 20 second client limit.
- LLM requests include a bounded token cap so the stage does not wait on unnecessary free-form reasoning output.
- Invalid timeout configuration falls back to the safe default and configured timeout remains capped.
- ECS one-shot stage tasks receive `PIPELINE_DOMAIN_CANDIDATE_LLM_TIMEOUT_SECONDS` and related domain-candidate runtime settings.
- Review question enrichment defaults to a timeout longer than 30 seconds and ECS one-shot stage tasks can override it.
- Review question enrichment does not send a hardcoded client-side `max_tokens` value, while still disabling Qwen thinking in the request payload.
- A single review-question LLM request failure records `request_failure` fallback on that question and does not fail the whole `feedback_candidate_generation` stage.
- `stop_llm_service` skips replay cleanup during an initial pipeline run.
- Terraform defines the Airflow ECS to LLM ALB port 8000 security-group path.
- Terraform configures the internal LLM ALB idle timeout above the domain candidate LLM client timeout.
- Terraform does not reset the LLM ECS service `desired_count` while Airflow owns on-demand start/stop for a pipeline run.
