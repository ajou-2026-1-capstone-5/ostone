# 업로드 후 자동 파이프라인 상태 추적

## Goal

워크스페이스 상담 로그 업로드 화면에서 업로드 완료 이후 자동 생성된 Airflow pipeline job의 현재 상태를 같은 화면에서 확인할 수 있게 한다.

## Problem

상담 로그 업로드는 dataset 생성 후 ingestion pipeline job을 자동으로 생성할 수 있지만, 일반 업로드 화면에서는 업로드 직후 해당 job이 생성되었는지, 실행 중인지, 검토 화면으로 이동해야 하는지 확인하기 어렵다. 사용자는 별도 pipeline 화면이나 직접 생성 API 호출 여부를 추측해야 한다.

## Scope

- workspace member가 dataset 기준 최신 pipeline job 상태를 조회할 수 있는 backend API를 제공한다.
- 업로드 성공 후 `LogUploadForm`에서 최신 `INGESTION` job 상태를 조회하고 진행 중 상태를 polling한다.
- 자동 생성 job이 확인되면 업로드 완료 요약 아래에 상태, Airflow run, DAG, 실행 시간, 오류 메시지, 상태 화면 이동 액션을 표시한다.
- 자동 생성 job이 아직 없을 때만 기존 수동 도메인팩 생성 fallback을 제공한다.

## Non-goals

- Airflow DAG 실행 방식이나 callback payload contract를 변경하지 않는다.
- dataset upload API의 request/response contract를 변경하지 않는다.
- pipeline review 화면의 검토 UX를 변경하지 않는다.
- OpenAPI generated file을 직접 수정하지 않는다.

## Requirements

1. Backend는 `GET /api/v1/workspaces/{workspaceId}/datasets/{datasetId}/pipeline-jobs/latest` 요청을 제공해야 한다.
2. Backend는 workspace 존재 여부, 요청 사용자 membership, dataset의 workspace 소속을 검증해야 한다.
3. Backend는 지원하는 job type만 조회해야 하며 기본 조회 대상은 `INGESTION`이어야 한다.
4. Backend 응답은 pipeline job이 없는 경우에도 성공 응답으로 `pipelineJob: null`을 반환해야 한다.
5. Frontend는 업로드 성공 후 dataset id가 있으면 최신 ingestion job 상태를 조회해야 한다.
6. Frontend는 job 상태가 `SUCCEEDED`, `FAILED`, `CANCELLED`가 되기 전까지 주기적으로 상태를 갱신해야 한다.
7. Frontend는 자동 job 상태 패널을 업로드 완료 요약 바로 아래에 표시해야 한다.
8. Frontend는 자동 job이 확인되지 않은 경우에만 수동 도메인팩 생성 fallback을 유지해야 한다.

## Data/API Impact

- 신규 read-only API가 추가된다.
- 기존 upload API, domain-pack generation trigger API, pipeline review API의 contract는 변경하지 않는다.
- Frontend manual API allowlist에는 OpenAPI generated endpoint가 아직 없는 최신 pipeline job 조회 wrapper만 등록한다.

## Acceptance Criteria

- 업로드 완료 후 자동 ingestion job이 있으면 업로드 화면에서 `자동 생성 파이프라인` 상태 패널이 보인다.
- 상태 패널에서 job id, Airflow run id, DAG, 실행 시간 또는 상태 설명을 확인할 수 있다.
- 상태 패널의 주요 액션으로 pipeline job 상태/검토 화면으로 이동할 수 있다.
- 최신 ingestion job이 없거나 조회 중일 때는 준비/조회 상태가 표시된다.
- 조회 실패 시 사용자는 같은 화면에서 상태 새로고침을 실행할 수 있다.
- 자동 job이 확인되면 수동 도메인팩 생성 CTA가 중복 노출되지 않는다.

## Validation

- `GetLatestPipelineJobUseCaseTest`로 membership, dataset 소속, job type, no-job 응답을 검증한다.
- `PipelineJobStatusControllerTest`로 인증 사용자 기준 latest status API 응답을 검증한다.
- `LogUploadForm.test.tsx`로 업로드 성공 후 자동 pipeline 상태 패널과 상태 화면 이동을 검증한다.
- Frontend manual API audit가 신규 wrapper allowlist를 통과하는지 확인한다.
