# #940 상담 도메인 확정 화면 추천 도메인 중복 순위 오류 수정

## 배경 / 문제

상담 도메인 확정 화면(`PipelineReviewCheckpointCard`, Domain confirmation)에서 추천 도메인 후보들의
순위가 모두 달라야 하는데, 서로 다른 후보가 같은 순위(예: 둘 다 "2순위")로 표시되는 문제가 보고됨(이슈 #940).

- 하나카드 데모 사용자 계정 화면에서 재현됨
- 순위 표시로 바뀌기 이전, 정확도 퍼센티지로 보여줄 때는 후보별 값이 전부 유의미하게 달랐음

## 원인 분석

순위 라벨을 만드는 `formatDomainConfidence`가 후보 간 상대 비교가 아니라
**고정 임계 구간(absolute threshold)** 으로 순위를 매기고 있었다.

```
confidence >= 0.8 → 1순위
confidence >= 0.5 → 2순위
그 외            → 3순위
```

이 때문에 confidence 값이 서로 달라도 같은 구간에 들어가면(예: 0.62, 0.55) 동일하게 "2순위"로
표시되어 중복 순위가 발생한다. 퍼센티지로 보여줄 때는 값이 그대로 노출되어 문제가 드러나지 않았으나,
순위 라벨로 압축되면서 구간 묶임이 중복 순위로 표면화되었다.

## 해결 방안

절대 임계값이 아니라 **후보 간 상대 순위**(높은 confidence가 1순위)로 순위를 매긴다.

- `domainRankByTaskId`(useMemo) 추가: confidence가 숫자인 후보를 내림차순 정렬해 task id → 순위 맵 생성
- dense ranking 적용: confidence가 정확히 같은 후보만 같은 순위를 공유하고, 값이 다르면 순위가 항상 갈린다
- confidence가 없는 fallback 후보는 순위에서 제외(기존처럼 원인 badge로 구분)
- `formatDomainConfidence` → `formatDomainRank(rank)` 로 교체하고,
  후보 카드 배지 / 선택된 후보 신뢰도 표시 두 곳을 순위 맵 기반으로 변경

## 검증

- 같은 구간(0.5~0.8)에 속하는 서로 다른 confidence(0.62 / 0.55 / 0.88) 후보 3개가 1·2·3순위로 모두 구분됨
- 단일 후보는 그대로 1순위로 표시됨(기존 동작 유지)
- `PipelineReviewCheckpointCard` 테스트 전체 통과, tsc / ESLint 클린
