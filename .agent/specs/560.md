# 도메인팩 상세에서 Slot/Policy/Risk 진입성 명확화

## Goal

도메인팩 상세 요약에서 Slot, Policy, Risk 화면이 실제로 존재하고 진입 가능하다는 점을 운영자가 즉시 알 수 있게 한다. 각 구성요소 카드는 목록 CTA와 미리보기 상세 진입을 일관되게 제공한다.

## Problem

도메인팩 상세에는 Slot, Policy, Risk 라우트와 화면이 존재하지만, 요약 카드에서 화면 존재와 이동 경로가 충분히 드러나지 않으면 기능이 없는 것처럼 보일 수 있다. 특히 Risk 카드는 요약 구성요소에 노출되지 않아 Slot/Policy/Workflow와 같은 발견 경로가 없다.

## Non-Goals

- Backend API, schema, generated API 파일을 변경하지 않는다.
- Slot, Policy, Risk 생성/수정 권한 모델을 변경하지 않는다.
- Intent/Workflow의 기존 상세 진입 흐름을 제거하거나 약화하지 않는다.

## Current Contract

| 영역                   | 현재 상태                                                                       |
| ---------------------- | ------------------------------------------------------------------------------- |
| 도메인팩 요약 구성요소 | Intent, Slot, Policy, Workflow 카드를 표시하고 Risk 카드는 표시하지 않음        |
| 카드 CTA               | 헤더 화살표 버튼 중심이라 목록 이동 가능성이 명시적으로 보이지 않음             |
| 미리보기 상세 진입     | Slot/Workflow는 미리보기 항목 상세 진입이 있고 Policy/Risk는 일관되지 않음      |
| 빈/미선택 상태         | 단순히 선택 또는 초안 없음만 안내해 화면 존재를 덜 드러냄                       |
| Policy/Risk 목록-상세  | 데스크톱 폭에서도 목록과 상세가 분리 표시되어 Slot과 다른 패턴으로 보일 수 있음 |

## Requirements

1. 도메인팩 상세 구성요소 영역에 Risk 카드를 추가한다.
2. Intent, Slot, Policy, Risk, Workflow 카드에 명시적인 `목록 보기` CTA를 제공한다.
3. 가능한 미리보기 항목은 해당 상세 경로로 직접 이동할 수 있어야 한다.
4. 미리보기 데이터가 없거나 상세 선택이 없을 때도 목록/상세 화면의 존재를 알 수 있는 안내 문구를 표시한다.
5. Slot, Policy, Risk 목록-상세 화면은 데스크톱에서 같은 2-pane 진입 패턴을 유지한다.
6. 기존 Intent/Workflow 경로와 버전 query parameter 전달을 유지한다.

## Acceptance Criteria

- [ ] 사용자는 도메인팩 상세에서 Slot, Policy, Risk 목록으로 쉽게 이동할 수 있다.
- [ ] 각 카드/CTA가 클릭 가능하다는 affordance를 명확히 가진다.
- [ ] 목록이 비어 있어도 화면이 없다고 오해하지 않는 empty state가 표시된다.
- [ ] Slot, Policy, Risk의 목록-상세 레이아웃과 내비게이션이 일관된다.
- [ ] 기존 Intent/Workflow 진입 흐름과 충돌하지 않는다.
