# [BE-860] 하나카드 데모 도메인팩 seed를 1000건 워크플로우로 확장

> **Backlog**: 하나카드 카드 상담 데모 도메인팩이 더 많은 상담 유형을 커버하도록 시드 워크플로우를 확장하고, 시드 콘텐츠가 바뀌면 재기동 시 새 published 버전으로 반영되게 한다.
> **Layer**: Backend
> **Branch**: `feature/860-hanacard-1000-workflows`
> **Verified existing paths**: `backend/src/main/java/com/init/domainpack/infrastructure/DomainPackSeedRunner.java`, `backend/src/main/resources/seed/hanacard-workflow-candidate.json`, `backend/src/test/java/com/init/domainpack/infrastructure/DomainPackSeedRunnerTest.java`

---

## Goal

하나카드 데모 도메인팩 시드를 `hanacard_100` 기반에서 `hanacard_1000` 상담 로그 기반으로 확장(워크플로우 1000건)하고, 시드 콘텐츠가 변경되면 backend 재기동 시 멱등성을 유지하면서도 변경분이 새 published 버전으로 반영되게 한다.

---

## Current State

- 하나카드 시드(`hanacard-workflow-candidate.json`)는 `hanacard_100` 로그 기반의 소규모 워크플로우 집합이었다.
- `DomainPackSeedRunner`(시드 확장 전 명칭 `ActiveVentureDomainPackSeedRunner`)는 이미 published 버전이 존재하면 **항상 기존 버전에 backfill만** 수행했다. 따라서 시드 JSON 본문을 확장해도 이미 시드된 환경에서는 새 워크플로우가 반영되지 않았다(기존 버전 그대로 유지).

---

## Target Flow

1. **시드 콘텐츠 확장**: `hanacard-workflow-candidate.json`을 `hanacard_1000` 상담 로그에서 생성한 1000건 워크플로우로 교체한다. `SeedConfig` description을 `hanacard_1000 ...`로 갱신한다.
2. **재실행 분기 개선**: 현재 published 버전이 있을 때 시드의 `summaryJson`과 현재 버전의 summary를 비교한다.
   - **동일**(`hasSameSeedSummary`) → 기존 버전에 내부 리소스/슬롯명/바인딩 프롬프트 backfill만 수행(`backfillExistingPublishedVersion`).
   - **변경** → 새 published 버전을 생성(`persistNewPublishedVersion`)해 확장된 시드를 반영한다.
3. 기존 멱등성(중복 version 미생성)·prod에서 matching profile build enqueue 스킵 동작은 유지한다.

> 참고: 본 브랜치에는 위 시드 확장과 함께, 두 도메인팩(액티벤처·하나카드)을 모두 시드하는 러너의 명칭을 실제 책임에 맞게 `ActiveVentureDomainPackSeedRunner` → `DomainPackSeedRunner`로 정리하는 리네임이 포함된다(동작 변경 없음).

---

## Acceptance

- [ ] 깨끗한 DB에서 `local`/`dev`/`prod` 기동 시 하나카드 도메인팩이 1000건 워크플로우로 published 시드된다.
- [ ] 동일 시드로 재기동 시 새 version이 생성되지 않고 backfill만 수행된다(멱등성).

- [ ] `prod` 프로파일에서는 matching profile build enqueue가 스킵된다.
- [ ] `DomainPackSeedRunnerTest` 통과 및 `./gradlew spotlessCheck` 통과.
