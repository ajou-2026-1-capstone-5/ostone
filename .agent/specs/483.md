# Backend Bedrock Chat Startup Fix

## Context

Production backend startup must use Bedrock Converse for chat generation while local/default
development can continue using the OpenAI-compatible chat configuration. The same change set also
keeps Airflow startup ordering and AWS SDK dependency alignment stable for production deployment.

## Requirements

- Production profile selects the Bedrock Converse chat provider without requiring an OpenAI API key.
- Default profile keeps the OpenAI-compatible chat provider explicit so multiple Spring AI chat
  model auto-configurations do not break application context startup.
- Backend `ChatClient` construction chooses the configured chat model deterministically when both
  OpenAI and Bedrock Converse starters are present.
- Bedrock embedding defaults use Cohere Embed v4 in the Seoul region and request vectors compatible
  with the existing `vector(1024)` schema.
- AWS SDK dependencies resolve through one current BOM version, without older hard-pinned AWS SDK
  artifacts in the lockfile.
- Airflow ECS startup keeps the metadata migration and Simple Auth password generation available to
  the serving tasks.
